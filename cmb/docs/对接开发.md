更新时间：2025-05-26 10:22:34

## 文档说明
本文档重点介绍招商银行新直联（免前置）开发对接流程，包含密钥生成、请求处理、报文规范、并发限制等核心内容，适用于开发者对接招商银行银企直联接口。

## 1. 并发限制
- 接口调用并发数限制：单用户（UID）同时请求数不超过**5个**
- 单接口TPS限制：默认单接口每秒请求数不超过**10次**
- 超出并发限制处理：服务器会返回限流错误码（具体错误码见附录），建议对接方做请求排队或重试机制

## 2. 报文规范
### 2.1 请求报文
#### 基本结构
请求报文分为`request`和`signature`两部分，其中：
- `request`：业务请求体，包含`head`（请求头）和`body`（业务参数）
- `signature`：签名信息，包含`sigtim`（签名时间）和`sigdat`（签名数据）

#### 请求头（head）必填字段
| 字段名   | 说明                                      | 示例值                              |
|----------|-------------------------------------------|-------------------------------------|
| funcode  | 接口功能码                                | DCPAYOPR                            |
| userid   | 客户用户ID（UBANK银企直联页面可查）       | N002432758                          |
| reqid    | 请求唯一标识（建议规则：时间戳+随机串）   | 201901091123456780001fbdev01        |

#### 业务参数（body）
根据不同`funcode`的接口要求填写，示例参考4.2节。

### 2.2 返回报文
#### 基本结构
返回报文分为`response`和`signature`两部分，其中：
- `response`：业务返回体，包含`head`（返回头）和`body`（业务返回数据）
- `signature`：签名信息，格式同请求报文

#### 返回头（head）核心字段
| 字段名     | 说明                                      | 示例值                              |
|------------|-------------------------------------------|-------------------------------------|
| funcode    | 对应请求的功能码                          | DCPAYOPR                            |
| reqid      | 对应请求的reqid                           | 201901091123456780001fbdev01        |
| resultcode | 处理结果码（000000为成功）                | FAAQ086                             |
| resultmsg  | 处理结果描述                              | CSAB051 户口110902773110203不存在    |
| rspid      | 响应唯一标识                              | 201908231436248870001001FBDEV01KF   |
| userid     | 客户用户ID                                | N002432758                          |

## 3. 前置客户端对接开发流程
> 注：前置客户端对接为传统模式，目前推荐使用免前置对接（4节），本章节仅做简要说明
1. 下载并部署招商银行前置客户端
2. 配置客户端参数（银行端提供配置文件）
3. 本地系统与前置客户端进行Socket/HTTP通信
4. 前置客户端完成加密、签名、与银行服务端交互
5. 前置客户端返回处理结果给本地系统

## 4. 免前置对接开发流程
### 4.1 密钥生成
#### 密钥类型
- 非对称密钥：SM2算法（签名/验签），包含公钥和私钥
- 对称密钥：SM4算法（加密/解密），16位长度

#### 生成流程
1. 对接方生成SM2密钥对（私钥自行保管，公钥提交银行备案）
2. 银行分配SM4对称密钥（线下或接口获取，需加密存储）
3. 国密算法向量参数：长度16位，取值为用户ID，不足16位右侧补0

#### 步骤 5：发送请求
##### 发送格式
以表单形式（application/x-www-form-urlencoded）POST 发送，参数格式：
```text
UID=N002432758&FUNCODE=DCPAYOPR&ALG=SM&DATA=加密后的BASE64字符串
```

##### 参数说明
| 参数名  | 说明                                       | 必填 | 示例值                               |
| ------- | ------------------------------------------ | ---- | ------------------------------------ |
| UID     | 客户用户 ID（与报文中 userid 一致）        | 是   | N002432758                           |
| FUNCODE | 接口功能码（与报文中 funcode 一致）        | 是   | DCPAYOPR                             |
| ALG     | 加密算法类型（默认 SM，国密算法）          | 是   | SM                                   |
| DATA    | 加密后的报文（需 URLEncode 处理）          | 是   | 上述 SM4 加密后的 BASE64 字符串     |

##### 发送地址
- 测试环境：http://cdctest.cmburl.cn/cdcserver/api/v2
- 生产环境：https://cdc.cmbchina.com/cdcserver/api/v2
##### 注意事项
- 请求发送后服务器同步返回结果，非 200 状态码也需读取返回内容（含报错信息）
- DATA 参数必须做 URLEncode 处理，避免特殊字符导致解析失败
#### 步骤 6：接收并处理返回结果
##### 解密
1. 接收银行返回的加密字符串（BASE64 编码）
2. 使用 SM4 对称密钥进行 SM4_CBC 解密（参数同加密）
3. 得到解密后的返回报文
##### 验签
1. 提取返回报文中的sigdat值并保存
2. 将报文中的sigdat替换为占位符__signature_sigdat__
3. 对替换后的报文按 ASCII 码排序、去空格换行，生成待验签字符串
4. 使用招商银行 SM2 公钥对sigdat值和待验签字符串进行 SM2 验签
5. 验签通过则确认报文未被篡改，否则视为无效报文

## 5. 示例代码
注：示例代码仅展示核心逻辑，需结合实际开发语言（Java/Python/PHP 等）调整，完整代码可联系银行对接专员获取
### 5.1 Python 示例（核心逻辑）
```python
import base64
import json
from gmssl import sm2, sm4

# 1. 配置密钥
SM2_PRIVATE_KEY = "对接方SM2私钥"  # 自行生成
SM2_PUBLIC_KEY_BANK = "招行SM2公钥"  # 银行提供
SM4_KEY = "招行分配的16位SM4密钥"  # 16位字符串
USER_ID = "N002432758"

# 2. 预添加signature
def add_signature(original_data, sigtim):
    original_data["signature"] = {
        "sigtim": sigtim,
        "sigdat": "__signature_sigdat__"
    }
    return original_data

# 3. 生成待签名字符串（排序+去空格）
def get_sign_str(data):
    # 递归排序字典KEY
    def sort_dict(d):
        if isinstance(d, dict):
            return {k: sort_dict(v) for k, v in sorted(d.items())}
        return d
    sorted_data = sort_dict(data)
    # 转为字符串并移除空格换行
    sign_str = json.dumps(sorted_data, separators=(',', ':'), ensure_ascii=False)
    return sign_str

# 4. SM2签名
def sm2_sign(sign_str, private_key):
    sm2_crypt = sm2.CryptSM2(private_key=private_key, public_key="")
    sign = sm2_crypt.sign(sign_str.encode('utf-8'))
    return base64.b64encode(sign).decode('utf-8')

# 5. SM4加密
def sm4_encrypt(data_str, key, iv):
    sm4_crypt = sm4.CryptSM4()
    sm4_crypt.set_key(key.encode('utf-8'), sm4.SM4_ENCRYPT)
    encrypt_data = sm4_crypt.crypt_cbc(iv.encode('utf-8'), data_str.encode('utf-8'))
    return base64.b64encode(encrypt_data).decode('utf-8')

# 6. 主流程
if __name__ == "__main__":
    # 原始报文
    original_data = {
        "request": {
            "head": {"funcode": "DCPAYOPR", "userid": USER_ID, "reqid": "201901091123456780001fbdev01"},
            "body": {"modnbr": "000002", "refext": "2019012316273819", "sndeac": "110902773110203",
                     "rcveac": "110902773110402", "rcvean": "北京全聚德", "trsamt": "100.99"}
        }
    }
    # 步骤1：添加signature
    sigtim = "20190823093102"
    data_with_sig = add_signature(original_data, sigtim)
    # 步骤2：生成待签名字符串
    sign_str = get_sign_str(data_with_sig)
    # 步骤3：SM2签名
    sigdat = sm2_sign(sign_str, SM2_PRIVATE_KEY)
    data_with_sig["signature"]["sigdat"] = sigdat
    # 步骤4：SM4加密
    data_str = json.dumps(data_with_sig, ensure_ascii=False)
    iv = USER_ID.ljust(16, '0')[:16]  # 向量：用户ID补0到16位
    encrypt_data = sm4_encrypt(data_str, SM4_KEY, iv)
    # 步骤5：构造请求参数
    request_params = {
        "UID": USER_ID,
        "FUNCODE": "DCPAYOPR",
        "ALG": "SM",
        "DATA": encrypt_data
    }
    print("请求参数：", request_params)
```

### 5.2 验签示例（Python）
```python
# SM2验签
def sm2_verify(sign_str, sigdat, public_key):
    sm2_crypt = sm2.CryptSM2(private_key="", public_key=public_key)
    sigdat_bytes = base64.b64decode(sigdat)
    return sm2_crypt.verify(sigdat_bytes, sign_str.encode('utf-8'))

# 解密返回报文并验签
def verify_response(encrypt_response, sm4_key, user_id, bank_sm2_pub):
    # SM4解密
    iv = user_id.ljust(16, '0')[:16]
    sm4_crypt = sm4.CryptSM4()
    sm4_crypt.set_key(sm4_key.encode('utf-8'), sm4.SM4_DECRYPT)
    decrypt_data = sm4_crypt.crypt_cbc(iv.encode('utf-8'), base64.b64decode(encrypt_response))
    response_data = json.loads(decrypt_data.decode('utf-8'))
    
    # 提取sigdat并替换占位符
    sigdat = response_data["signature"]["sigdat"]
    response_data["signature"]["sigdat"] = "__signature_sigdat__"
    
    # 生成待验签字符串
    verify_str = get_sign_str(response_data)
    # 验签
    verify_result = sm2_verify(verify_str, sigdat, bank_sm2_pub)
    return verify_result, response_data
```
## 附录
### 常见错误码
| 错误码   | 说明             | 处理建议                     |
| -------- | ---------------- | ---------------------------- |
| 000000   | 处理成功         | 正常解析业务数据             |
| FAAQ086  | 账户不存在       | 核对账户号是否正确           |
| FAAQ999  | 签名验证失败     | 检查签名流程、密钥是否正确   |
| FAAQ888  | 报文解密失败     | 检查 SM4 密钥、向量是否正确  |
| FAAQ777  | 请求超时 / 并发超限 | 降低并发量，增加重试机制     |

### 对接注意事项
1. 密钥需加密存储，禁止明文写在代码中
2. reqid 需保证全局唯一，避免重复请求导致业务异常
3. 生产环境需使用 HTTPS 协议，测试环境可使用 HTTP
4. 建议对接完成后进行全流程联调测试，银行提供测试账户
5. 接口调用日志需留存至少 6 个月，便于问题排查
