# 企银支付业务查询 BB1PAYQR

## 1. 使用场景

该接口用于支付业务提交之后，按照“业务参考号”对支付业务处理结果的查询，可以获得最终支付状态结果。

## 2. 使用说明

1.  **权限检查**：按业务参考号查询用户有权限的交易记录。注意登录用户要有查询权限，若未查询出结果，可优先检查是否有对应权限。避免由于权限问题查询不到业务结果，进而发生重复交易。
2.  **查询范围**：该接口支持查询含当前日期 13 个月以内的交易，超过这个日期范围不支持。
3.  **空结果处理**：查询成功，但是返回信息为空串，表示不存在任何匹配条件的交易，可能是银行没有收到客户请求或者超出查询时间范围。
4.  **状态判断方法**：
    *   返回的每笔信息中 `reqSts` 如果不等于 `FIN` 表示该笔支付银行还在处理中。
    *   `reqSts` = `FIN` 时再判断 `rtnFlg`：
        *   `rtnFlg` 为 `S` 时表示成功。
        *   当为 `F\B\R\D\C` 时，表示客户发起的这笔支付已经是终态了，且没有该笔款项没有成功支付到收款人账户，客户可以就该笔款项重新发起支付。
    *   **重发规则**：重发时不能改变业务参考号，非上述状态均不得重新发起支付。
5.  **频率限制**：推荐付款指令提交后 5 分钟发起结果同步，两次查询类请求最小调用间隔为 2s。
