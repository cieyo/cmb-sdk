# 账务查询 - 账户交易信息查询（trsQryByBreakPoint）
## 一、功能概述
用于查询本单位账户或集团公司全国范围内招商银行开户子公司账户的交易信息，支持近13个月内对公客户金融交易数据查询，也可申请查询13个月外至5年内数据（新增功能，不支持频繁查询）。

### 关键说明
1. 13个月内查询：直接对接接口即可；
2. 13个月外-5年内查询：需联系客户经理申请开通功能权限，默认有效期1年；
3. 交易数据处理规则：
    - 多级汇总：日间隐藏实时划拨交易，日终汇总后新增汇总交易；
    - 组合存款：协议内交易屏蔽，支取类仅体现利息金额，关联/取消关联视为一笔交易；
4. 断点续传：单次最多查询200条记录，支持循环查询直至数据完整；
5. 户口号与账号：一个户口号（cardNbr）可能对应多个币种的账号（acctNbr），查询时传入户口号即可，账号为系统内部续传使用；
6. 响应报文：尾部空格可自行去除。

## 二、请求报文
### 整体结构
| 字段名称 | 字段ID | 数据类型 | 是否必输 | 描述 |
|----------|--------|----------|----------|------|
| 输入接口 | TRANSQUERYBYBREAKPOINT_X1 | JSONArray | Y | 单记录查询核心参数 |
| 输入接口 | TRANSQUERYBYBREAKPOINT_Y1 | JSONArray | N | 多记录续传/断点查询参数，首次查询非必传 |

### TRANSQUERYBYBREAKPOINT_X1（单记录）
| 字段名称 | 字段ID | 数据类型 | 是否必输 | 描述 |
|----------|--------|----------|----------|------|
| 户口号 | cardNbr | String(35) | Y | - |
| 开始日期 | beginDate | D | Y | 查询交易时间段的开始日期 |
| 结束日期 | endDate | D | Y | 查询交易时间段的结束日期 |
| 起始记账序号 | transactionSequence | String(9) | 可选 | 不传入续传键值时有效，默认从第1笔开始 |
| 币种 | currencyCode | String(2) | 可选 | 建议传入 |
| 继续查询账号 | queryAcctNbr | String(200) | 可选 | 首次查询无需传，续传时赋值响应报文Z1中的对应字段 |
| 保留字段 | reserve | String(200) | 可选 | 字段长度未限制 |
| 借贷码 | loanCode | String(1) | 可选 | C=贷方，D=借方 |

### TRANSQUERYBYBREAKPOINT_Y1（多记录）
| 字段名称 | 字段ID | 数据类型 | 是否必输 | 描述 |
|----------|--------|----------|----------|------|
| 账号 | acctNbr | String(200) | - | - |
| 交易日期 | transDate | D | - | 当前查询最后一笔交易日期 |
| 期望下一记账序号 | expectNextSequence | String(9) | - | 期望下一笔记账的序号 |

### 4.断点查询请求范例

{
    "request": {
        "body": {

                    "cardNbr": "755947919810515",

                    "endDate": "20230502",
                    "transactionSequence": "1",
                    "currencyCode": "",

                    "reserve": ""

                {
                    "acctNbr": "755947919880003",
